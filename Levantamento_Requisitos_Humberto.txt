Extensão de Requisitos - APEX GOVERNANCE (Fase 2: Aprimoramento)
1. Requisitos Funcionais (RF) - Novos Recursos
Módulo 6: Visualização de Dados Avançada
[RF021] Dashboard Gráfico de Evolução: O sistema deve exibir um gráfico de linha mostrando a tendência histórica das métricas de qualidade (ex: "Evolução da Completude nos últimos 30 dias").

Justificativa: Permite aos gestores visualizar se as ações de correção estão surtindo efeito ao longo do tempo.

[RF022] Gráficos de Distribuição (Drill-down): O sistema deve apresentar gráficos de pizza ou barras para categorizar os dados (ex: "Distribuição de Sistemas Operacionais" ou "Usuários Inativos por Departamento").

Justificativa: Facilita a identificação rápida de ofensores e agrupamentos de problemas.

Módulo 7: Acionabilidade (Actionability)
[RF023] Exportação de Falhas (CSV/Excel): Nas telas de detalhes de relatórios, deve haver uma opção para baixar apenas os registros que falharam na validação em formato CSV.

Justificativa: Permite que o auditor envie a lista exata de erros para as equipes responsáveis (RH ou Suporte) realizarem a correção manual.

[RF024] Notificações Automáticas: O sistema deve ser capaz de enviar um e-mail semanal para os administradores com um resumo executivo das métricas de qualidade.

Justificativa: Garante a visibilidade passiva dos problemas, sem que o usuário precise acessar o painel ativamente.

Módulo 8: Segurança e Controle de Acesso (RBAC)
[RF025] Autenticação de Usuário: O sistema deve exigir login (usuário e senha) para acessar qualquer rota, exceto a página de login.

[RF026] Perfis de Acesso (RBAC): O sistema deve diferenciar permissões:

Visualizador: Apenas vê dashboards e relatórios.

Auditor: Pode executar análises.

Admin: Pode executar o Pipeline de Carga (ETL) e gerenciar configurações.

[RF027] Rastreabilidade de Ações: O sistema deve registrar qual usuário solicitou cada execução de pipeline ou análise (campo usuario_solicitante nos models).

Justificativa: Auditoria do próprio sistema de governança (quem fez o quê e quando).

Módulo 9: Interface do Usuário (UI/UX)
[RF028] Navegação Persistente: O sistema deve possuir uma barra lateral (sidebar) ou menu superior fixo para navegação rápida entre Carga, Análises Simples e Análises Relacionais.

[RF029] Feedback Visual de Processamento: Ao clicar em botões de ação, o botão deve entrar em estado de "carregando" (spinner) ou ser desabilitado para evitar cliques duplos.

2. Requisitos Não Funcionais (RNF) - Melhorias de Qualidade
Performance e Escalabilidade
[RNF006] Paginação de Resultados: As telas de detalhes não devem carregar mais de 50 registros por página. Deve haver controles de "Próximo/Anterior".

Justificativa: Previne travamento do navegador ao tentar renderizar tabelas com milhares de linhas de erro.

[RNF007] Indexação de Banco de Dados: As colunas utilizadas frequentemente em filtros (manager, department, sAMAccountName, objectGUID) devem possuir índices B-Tree no PostgreSQL.

Justificativa: Acelera drasticamente as consultas SQL das regras de negócio relacionais.

Interface e Usabilidade
[RNF008] Framework de Design: A interface deve ser padronizada utilizando um framework CSS moderno (Bootstrap 5 ou Tailwind), garantindo responsividade e consistência visual.

Infraestrutura e DevOps
[RNF009] Containerização (Docker): O sistema deve possuir um arquivo docker-compose.yml que orquestre a criação dos containers de Aplicação (Django), Worker (Celery), Broker (Redis) e Banco (PostgreSQL).

Justificativa: Facilita a implantação em qualquer servidor (On-premise ou Nuvem) e garante que o ambiente de desenvolvimento seja idêntico ao de produção.
